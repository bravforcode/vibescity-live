    **Part A — Defer GeoJSON in useMapLayers.js:**

    The `addNeonRoads()` function currently calls `map.value.addSource(...)` synchronously on `style.load`. This blocks the map ready signal while the browser parses 3.9MB of GeoJSON.

    Wrap the source+layer additions inside a `map.value.once('idle', () => { ... })` callback. The guard `if (map.value.getSource("neon-roads")) return;` must remain OUTSIDE the callback (checked before registering the idle listener) to prevent duplicate registrations.

    Updated `addNeonRoads` structure:
    ```js
    const addNeonRoads = () => {
      if (!map.value) return;
      if (map.value.getSource("neon-roads")) return; // already added

      map.value.once('idle', () => {
        if (!map.value) return;
        if (map.value.getSource("neon-roads")) return; // double-check after idle
        try {
          map.value.addSource("neon-roads", {
            type: "geojson",
            data: "/data/chiangmai-main-roads-lanes.geojson",
          });
          map.value.addLayer({ ...neon-roads-outer layer... });
          map.value.addLayer({ ...neon-roads-inner layer... });
        } catch (e) {
          console.warn("Neon roads setup failed:", e);
        }
      });
    };
    ```
    Keep the exact same layer definitions (id, type, source, paint) — only wrap them in `once('idle', ...)`.
